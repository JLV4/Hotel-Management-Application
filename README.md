# Hotel-Management-Application Project Description
This project involves building a hotel management application using Python and a PostgreSQL database to support two user roles: managers and clients. Managers can add or delete rooms and clients. They can retrieve statistics like top clients by bookings, room booking counts, and hotel averages. They can also generate cross‑city client lists, find problematic Chicago hotels, and report total client spending. Clients can register with names, emails, multiple addresses and credit cards. They can search for available rooms by date range, book a specific room, or use automatic booking by hotel. Clients can view their bookings and submit reviews only for hotels where they have stayed. The technical implementation uses a relational schema translated into a SQL script with Postgres dialect. Python handles business logic and embedded SQL queries for availability checks, booking insertion, and all manager or client reports. Data constraints include unique booking IDs, a rating range of 0 to 10, no overlapping bookings, and login via SSN for managers or email for clients.
